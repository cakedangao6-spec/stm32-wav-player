# 项目名称（简历版）

基于 STM32F103 的 WAV 音频存储与 PWM 播放系统

# 项目简介

基于 STM32 标准外设库实现 WAV 文件串口下载、W25Q64 SPI Flash 存储、WAV 头解析与 PWM 音频播放。系统通过 OLED 显示下载/播放状态，并支持串口命令交互与 CRC32 文件校验。

# 项目整体架构与数据流

电脑端通过 USART1 发送 `WRITE:size`、WAV 原始数据和 `VERIFY` 命令；STM32 使用 USART 接收中断将数据写入 Ring Buffer，主循环状态机按 W25Q64 页大小写入 SPI Flash，并通过 CRC32 校验写入结果。播放时，STM32 从 W25Q64 读取 WAV 文件，解析 RIFF/WAVE/fmt/data 信息，将 PCM 数据转换为 PWM 占空比，使用 TIM2 采样率中断驱动 TIM3_CH3 输出音频。

数据流：PC 串口 -> USART1 中断 -> Ring Buffer -> 文件传输状态机 -> SPI1 -> W25Q64 -> WAV 解析 -> 双缓冲 PCM/PWM 转换 -> TIM2 中断 -> TIM3 PWM 输出 -> 喇叭。

# 项目职责 / 项目内容

- 基于 STM32F10x 标准外设库完成 USART1、SPI1、GPIO、TIM2/TIM3、OLED 等外设初始化，搭建 WAV 下载与播放主流程。
- 实现 USART1 中断接收与 Ring Buffer 缓冲机制，将串口接收和主循环 Flash 写入解耦，提升连续文件传输稳定性。
- 实现 W25Q64 SPI Flash 驱动，支持 JEDEC ID 读取、扇区擦除、页编程、连续读取及忙状态等待。
- 设计串口文件传输状态机，支持 `PING`、`WRITE:size`、`VERIFY` 命令，完成 WAV 文件写入和 CRC32 校验反馈。
- 实现 WAV 文件头解析，支持 PCM 格式下 8/16 bit、单/双声道 WAV，提取采样率、位深、声道数和 PCM 数据偏移。
- 实现基于双缓冲的 PWM 音频播放：主循环从 W25Q64 填充空闲缓冲区，TIM2 中断按采样率切换样本并更新 TIM3_CH3 PWM 占空比。

# 技术栈

STM32F103C8T6、Keil5、ARMCC、STM32 标准外设库、C 语言、USART、SPI、W25Q64、Ring Buffer、CRC32、TIM2 定时中断、TIM3 PWM、OLED 软件 I2C。

# 技术亮点核查

已实际实现：Ring Buffer、SPI Flash 驱动、W25Q64 文件写入/读回、CRC32 校验、WAV 头解析、PWM 音频输出、播放双缓冲、OLED 状态显示、串口命令协议。

不应写入简历：DMA 音频播放、DAC 音频输出、MP3 解码、文件系统/FATFS、SD 卡播放、按键菜单、多歌曲列表、RTOS、多任务调度。这些功能当前代码中没有完整实现或不是主播放链路。
